

##  Routes

We will build a portfolio website with these routes:

what you need after installing the project is npm install react-router

and

import {
  createBrowserRouter,
  RouterProvider,
  NavLink,
  Outlet,
} from "react-router";


createBrowserRouter creates the router.
RouterProvider uses that router.
NavLink is better than a normal <a> tag because it does not reload the whole page.
Used inside a layout to show the selected child page.

| Import                | Use                                          |
| --------------------- | -------------------------------------------- |
| `createBrowserRouter` | Creates the route configuration              |
| `RouterProvider`      | Gives the route configuration to the app     |
| `NavLink`             | Creates navigation links with active styling |
| `Outlet`              | Shows child routes inside a layout           |


```txt
/
 /about
 /projects
 /skills
 /contact
 /not-found
```

Meaning:

```txt
/          Home page
/about     About page
/projects  Projects page
/skills    Skills page
/contact   Contact page
/wrong-url 404 Not Found page
```

# 1. What You Must Understand First

React has components, not pages.

A component becomes a page when a route connects a URL to it.

Example:

```jsx
{
  path: "about",
  element: <About />,
}
```

This means:

When the user visits `/about`, show the `About` component.

```txt
  layouts/
    RootLayout.jsx
```

Important:

Do not put all pages in `main.jsx`.

`main.jsx` is only the starting point of the app.

---

# 4. What `main.jsx` Should Do

Open `src/main.jsx`.

It should look like this:

```jsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import "./index.css";
import App from "./App.jsx";

createRoot(document.getElementById("root")).render(
  <StrictMode>
    <App />
  </StrictMode>
);
```

Explanation:

```jsx
import App from "./App.jsx";
```

This imports the main app.

```jsx
createRoot(document.getElementById("root")).render(...)
```

This tells React to display the app inside the `root` div in `index.html`.

`main.jsx` should not contain routes.
`main.jsx` should not contain page content.
`main.jsx` should not display all components.

Bad example:

```jsx
<Home />
<About />
<Projects />
<Skills />
<Contact />
```

Do not do this in `main.jsx`.

That would show all pages at the same time, which is not routing.

# 6. Create Navbar

Create `src/components/Navbar.jsx`.

```jsx
import { NavLink } from "react-router";

export default function Navbar() {
  const navStyle = ({ isActive }) => {
    return {
      fontWeight: isActive ? "bold" : "normal",
      color: isActive ? "red" : "black",
    };
  };

  return (
    <nav>
      <NavLink to="/" style={navStyle}>
        Home
      </NavLink>{" | "}

      <NavLink to="/about" style={navStyle}>
        About
      </NavLink>{" | "}

      <NavLink to="/projects" style={navStyle}>
        Projects
      </NavLink>{" | "}

      <NavLink to="/skills" style={navStyle}>
        Skills
      </NavLink>{" | "}

      <NavLink to="/contact" style={navStyle}>
        Contact
      </NavLink>
    </nav>
  );
}
```

Important:

`NavLink` is used for navigation links.

It can also detect which page is active.

# 7. Create Root Layout

Create `src/layouts/RootLayout.jsx`.

```jsx
import { Outlet } from "react-router";
import Navbar from "../components/Navbar";

export default function RootLayout() {
  return (
    <div>
      <Navbar />

      <main>
        <Outlet />
      </main>
    </div>
  );
}
```

Explanation:

The `Navbar` appears on every page.

The `Outlet` is where the selected page appears.

So if the user visits `/about`, the `About` page appears inside `Outlet`.

If the user visits `/skills`, the `Skills` page appears inside `Outlet`.

# 8. Create Routes in `App.jsx`

Open `src/App.jsx`.

```jsx
import {
  createBrowserRouter,
  RouterProvider,
} from "react-router";

import RootLayout from "./layouts/RootLayout";

import Home from "./pages/Home";
import About from "./pages/About";
import Projects from "./pages/Projects";
import Skills from "./pages/Skills";
import Contact from "./pages/Contact";
import NotFound from "./pages/NotFound";

const router = createBrowserRouter([
  {
    path: "/",
    element: <RootLayout />,
    children: [
      {
        index: true,
        element: <Home />,
      },
      {
        path: "about",
        element: <About />,
      },
      {
        path: "projects",
        element: <Projects />,
      },
      {
        path: "skills",
        element: <Skills />,
      },
      {
        path: "contact",
        element: <Contact />,
      },
      {
        path: "*",
        element: <NotFound />,
      },
    ],
  },
]);

export default function App() {
  return <RouterProvider router={router} />;
}
```

Explanation:

`createBrowserRouter` creates the route list.

`RouterProvider` activates the router inside the app.

This route:

```jsx
{
  index: true,
  element: <Home />,
}
```

means:

Show `Home` when the user is at `/`.

This route:

```jsx
{
  path: "about",
  element: <About />,
}
```

means:

Show `About` when the user is at `/about`.

This route:

```jsx
{
  path: "*",
  element: <NotFound />,
}
```

means:

Show `NotFound` for any unknown URL.

# 11. Important Teaching Point

The wrong way:

```jsx
// main.jsx
<Home />
<About />
<Projects />
<Skills />
<Contact />
```

This displays everything at once.

That is not routing.

The correct way:

```jsx
// main.jsx
<App />
```

Then:

```jsx
// App.jsx
<RouterProvider router={router} />
```

Then React Router decides which page to show.

# 14. Simple Summary

`main.jsx` starts the app.

`App.jsx` controls routing.

`RootLayout.jsx` provides shared layout.

`Navbar.jsx` provides navigation.

`pages/` contains the page components.

React Router chooses only one page to display based on the URL.

That is the whole point of routing.
